Release integrity

Release integrity

QSD Hive for Windows is currently published without Microsoft Authenticode signing. The SignPath Foundation application was declined for now because QSD needs more external public adoption signals before Foundation sponsorship is available.

Verify the SHA-256 checksum before installing. A checksum detects changed bytes; it does not make Windows show a verified publisher. SmartScreen may warn until QSD has a paid or accepted Authenticode signing path.

QSD-native ML-DSA-87 signatures authenticate release manifests and artifact hashes independently of Microsoft Authenticode trust.

Current Windows builds are unsigned. Trust the release only after the versioned download URL, SHA-256 checksum, and QSD-native release signature match the public manifest.
